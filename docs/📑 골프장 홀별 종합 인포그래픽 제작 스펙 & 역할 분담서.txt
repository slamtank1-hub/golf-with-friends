골프장 홀별 종합 인포그래픽 제작 최종 통합 스펙 v7

1. 제작 목적

나는 골프앱에 사용할 홀별 종합 인포그래픽 이미지를 만들고 있다.

각 홀마다 다음 3개의 원본 이미지를 제공한다.

1. 코스 탑뷰 이미지
2. 그린 등고선 / 그린 경사 이미지
3. 고도 단면도 / 종단 경사도 이미지

이 3개 이미지를 기반으로, 모든 홀에서 같은 형태와 같은 레이아웃을 가진 종합 이미지를 만든다.

중요한 점은 AI가 코스맵, 그린, 고도 그래프를 새로 상상해서 그리는 것이 아니라, 사용자가 제공한 원본 이미지를 최대한 그대로 사용하고, 그 위에 제목, 거리 정보, 나침반, 말풍선, 경사 화살표, 범례 등을 합성하는 방식이어야 한다.


2. 핵심 원칙

반드시 지켜야 할 핵심 원칙은 다음과 같다.

- AI는 코스맵이나 지형을 새로 상상해서 그리지 않는다.
- 원본 이미지의 형태, 벙커, 해저드, 카트길, 그린 레이아웃을 그대로 유지한다.
- 코스 탑뷰, 그린 등고선, 고도 단면도는 생성형으로 다시 그리지 않는다.
- 사용자가 제공한 원본 이미지를 그대로 삽입하고, 필요한 경우 crop, 마스킹, 밝기/대비 보정만 한다.
- 새로 추가하는 것은 제목, 거리 정보, 나침반, 말풍선, 경사 화살표, 범례, TIP 정도만 한다.
- 모든 홀의 패널 배치, 여백, 비율, 색상, 글자 스타일은 동일하게 유지한다.
- AI가 임의로 거리, 고도차, 홀 구조, 해저드 위치를 만들어내지 않는다.
- AI는 사용자가 표시한 흰색 원과 검은색 X를 삭제, 이동, 확대, 축소, 대체하지 않는다.
- AI는 코스뷰 위에 랜딩존 점선 원, 타깃 마크, 200m 텍스트, TEE 0m 라벨, 가이드라인을 추가하지 않는다.
- 랜딩존, 타깃, 200m 텍스트, 화이트 티 기준선은 사용자가 최종 이미지 완성 후 직접 추가한다.


3. 전체 캔버스 및 디자인 시스템

전체 캔버스:
- 크기: 1000 x 1414 px
- 세로형 Portrait
- 비율: 1 : 1.414

배경:
- 연한 아이보리 #F7F6F0

전체 외곽:
- 테두리: 1px solid #D9D7CF
- 외곽 라운드: 12px 고정

내부 패널:
- 코스맵 패널 라운드: 12px 고정
- 고도 단면도 패널 라운드: 12px 고정
- 그린 분석 패널 라운드: 12px 고정
- 티박스 거리 박스 라운드: 8px 고정
- 작은 라벨 / 말풍선 라운드: 10px 고정

라운드 처리 원칙:
- 라운드는 부드럽게 보이더라도 임의로 더 크게 키우지 않는다.
- 모든 홀에서 동일한 값을 사용한다.
- 특히 상단 헤더, 티박스 박스, 우측 분석 패널의 라운드는 항상 동일해야 한다.

기본 색상:
- 메인 다크그린 #0A2F1D
  - 제목
  - 기본 안내 박스
  - 전체 톤앤매너
- 추천 / 안전 / 공략 지점 #4E8752
  - 추천 구역
  - 안전 구역
  - 긍정적 플레이 요소
- 위험 / 절대 기피 / 주의 수치 #D93025
  - 위험 구역
  - 벌타 위험
  - 고도차 수치
- 경고 박스 #F2994A
  - 무리한 공략 방지
  - 치명적 위험 구역
- 패널 테두리 #D8D8D0
- 일반 텍스트 #111111

폰트:
- 기본 폰트: Noto Sans KR 또는 Pretendard 계열 고딕체
- 골프장/코스명: 24pt, Bold, 다크그린 #0A2F1D
- 홀 번호/PAR 정보: 66pt / 34pt 조합
- 섹션 제목: 14~16pt, Bold
- 말풍선/안내 문구: 11~12pt, Medium
- 범례/보조 문구: 9~10pt


4. 고정 레이아웃 구조

전체 화면은 항상 아래 구조를 유지한다.

[상단 헤더 영역]
- 골프장명 / 코스명
- 홀 번호 / PAR
- 티박스별 거리 탭
- 나침반

[좌측 코스맵 영역]
- 전체 폭의 약 55%
- 코스 전체 탑뷰 이미지 배치
- 사용자가 표시한 화이트 티 흰색 원과 검은색 X는 원본 그대로 보존
- AI는 랜딩존, 타깃, 200m 텍스트, TEE 0m 라벨, 기준선을 추가하지 않음
- 공략/위험 말풍선 표시
- 범례 표시

[우측 분석 영역]
- 전체 폭의 약 40%
- 세로 2단 구성
- 우측 상단: 고도 단면도
- 우측 하단: 그린 분석

기본 여백:
- 좌우 여백: 30~40px
- 패널 간격: 20~30px

중요:
- 홀마다 레이아웃을 바꾸지 않는다.
- 패널 위치, 크기, 여백, 범례 위치는 모든 홀에서 동일하게 유지한다.
- 정확한 픽셀 좌표는 AI가 디자인에 맞춰 조정할 수 있으나, 전체 구조와 비율은 변경하지 않는다.


5. 상단 헤더 영역 고정 스펙

상단 헤더는 절대 좌표를 기준으로 고정한다.

헤더 전체 영역:
- 좌표: x=24, y=24
- 크기: w=952, h=170
- 테두리: 1px solid #CFCBC1
- 라운드: 12px

AI는 코스명, 홀 번호, PAR, 티박스 거리 박스, 나침반을 이 헤더 영역 안에서만 배치한다.
각 요소의 위치는 아래 지정값으로 고정한다.


6. 코스명 영역

코스명 영역:
- 좌표: x=42, y=40
- 크기: w=360, h=28
- 정렬: 좌측 정렬
- 표기 형식: OAK VALLEY CC · PINE COURSE
- 폰트: Noto Sans KR 또는 Pretendard
- 크기: 24pt
- 굵기: Bold
- 색상: #0A2F1D

규칙:
- 코스명은 항상 헤더 좌측 상단에 배치한다.
- 줄바꿈하지 않는다.
- 가운데 정렬하지 않는다.
- 코스명 길이와 관계없이 시작 위치는 바꾸지 않는다.
- 표기상 COURSE 오타가 나지 않도록 한다. COURCE가 아니라 COURSE로 표기한다.


7. 홀 번호 / PAR 영역

홀 번호 / PAR 영역:
- 좌표: x=42, y=78
- 크기: w=300, h=74
- 정렬: 좌측 정렬
- 표기 형식: HOLE 2 | PAR 4

세부 텍스트 규격:

1) “HOLE {번호}”
- 크기: 66pt
- 굵기: ExtraBold
- 색상: #0A2F1D

2) 구분선 “|”
- 크기: 34pt
- 굵기: Medium
- 색상: #7D8B82

3) “PAR {숫자}”
- 크기: 34pt
- 굵기: Bold
- 색상: #356B3F

규칙:
- 홀 번호와 PAR은 항상 같은 줄에 배치한다.
- “HOLE {번호}”와 “PAR {숫자}”의 상대적 위치는 바꾸지 않는다.
- 홀 번호가 1자리/2자리여도 전체 그룹의 시작 위치는 고정한다.
- 홀 번호 길이 때문에 티박스 거리 박스 위치가 밀리면 안 된다.


8. 티박스 거리 박스 그룹

티박스 거리 박스 그룹:
- 그룹 시작 좌표: x=420, y=72
- 그룹 전체 크기: w=392, h=90
- 박스 수: 4개 고정
- 박스 간 간격: 8px
- 각 박스 크기: 92 x 90 px
- 라운드: 8px

각 박스 내부 구조:
- 상단 라벨 영역 높이: 34px
- 하단 거리 영역 높이: 56px
- 라벨 / 거리 모두 중앙 정렬

박스 순서:
1. 블랙
2. 블루
3. 화이트
4. 레이디/시니어

박스 스타일:

1) 블랙
- 라벨 배경: #111111
- 라벨 글자: #FFFFFF
- 박스 테두리: 1px solid #111111

2) 블루
- 라벨 배경: #0F4C9A
- 라벨 글자: #FFFFFF
- 박스 테두리: 1px solid #0F4C9A

3) 화이트
- 라벨 배경: #FFFFFF
- 라벨 글자: #111111
- 박스 테두리: 1px solid #BFC3C7

4) 레이디/시니어
- 라벨 배경: #FBEFF1
- 라벨 글자: #111111
- 박스 테두리: 1px solid #E9AEB7

폰트:
- 티박스 라벨: 16pt, Bold
- 거리 값: 24pt, ExtraBold
- 거리 표기 형식: 412m / 364m / 343m

규칙:
- 티박스 4개는 항상 한 줄로 배치한다.
- 박스 크기와 위치는 모든 홀에서 동일하게 유지한다.
- 거리 숫자 자릿수에 따라 박스 폭이나 위치를 바꾸지 않는다.
- 코스명, 홀 번호 길이 때문에 티박스 박스 위치를 이동하지 않는다.
- 거리 값은 반드시 숫자+m 형식으로 표기한다.


9. 나침반 재정의 스펙

중요:
기존처럼 “N 방향 자체를 회전시키는 방식”은 사용하지 않는다.
그 방식은 사용자가 제공한 방위각이 “북쪽의 방향”인지 “홀 진행 방향”인지 헷갈릴 수 있으므로 금지한다.

방위각 의미:
사용자가 제공하는 방위각 값은 “홀의 진행 방향, 즉 티에서 그린으로 향하는 방향이 북쪽을 기준으로 시계방향 몇 도인지”를 의미한다.

예시:
- 180도 = 홀 진행 방향이 정남향
- 150도 = 홀 진행 방향이 남동쪽 방향
- 90도 = 홀 진행 방향이 동쪽 방향
- 270도 = 홀 진행 방향이 서쪽 방향

나침반 표시 방식:
- 북쪽(N)은 항상 위쪽, 즉 12시 방향에 고정한다.
- 회전하는 것은 “홀 진행 방향 화살표” 1개뿐이다.
- 원형 나침반은 일반적인 북쪽 기준으로 고정한다.
- 사용자가 제공한 방위각만큼 홀 진행 방향 화살표만 시계방향으로 회전시킨다.

나침반 배치:
- 영역 좌표: x=850, y=36
- 영역 크기: w=104, h=120
- 원형 나침반 크기: 84 x 84 px
- 원형 나침반 중심: x=892, y=78
- 하단 각도 텍스트 영역: 84 x 24 px
- 각도 텍스트 위치: 원형 바로 아래 중앙

나침반 구성:
1) 원형 외곽선
- 선 두께: 3px
- 색상: #0A2F1D

2) N 문자
- 위치: 원 상단 중앙
- 크기: 18pt
- 굵기: Bold
- 색상: #0A2F1D
- 회전 금지

3) 홀 진행 방향 화살표
- 시작점: 원 중심
- 끝점: 방위각 방향
- 색상: #0A2F1D
- 명확한 화살촉 포함
- 사용자가 준 방위각만큼 시계방향 회전

4) 각도 숫자
- 표기 예: 150°
- 위치: 원 아래 중앙
- 크기: 18pt
- 굵기: Bold
- 색상: #0A2F1D

나침반 해석 예시:
- 방위각 150도:
  - N은 위에 고정
  - 홀 진행 방향 화살표는 오른쪽 아래 방향, 약 5시 방향
  - 하단 텍스트는 150°

- 방위각 180도:
  - 화살표는 정확히 아래, 6시 방향

- 방위각 90도:
  - 화살표는 오른쪽, 3시 방향

- 방위각 270도:
  - 화살표는 왼쪽, 9시 방향

절대 금지:
- N 글자를 회전하지 않는다.
- 나침반 전체를 돌리지 않는다.
- N이 아래쪽이나 옆쪽으로 가면 안 된다.
- 회전하는 것은 홀 진행 방향 화살표뿐이다.


10. 헤더 영역 고정 원칙

AI는 상단 헤더에서 아래 항목을 자동 재배열하면 안 된다.

절대 금지:
- 코스명 길이에 따라 티박스 위치를 밀기
- 홀 번호 자리수에 따라 PAR 위치를 밀기
- 티박스 거리 자릿수에 따라 박스 폭을 변경하기
- 나침반 위치를 임의로 이동하기
- 홀마다 라운드 값을 다르게 적용하기

반드시 지킬 것:
- 코스명, 홀/PAR, 티박스 4박스, 나침반은 항상 같은 자리
- 모든 홀에서 헤더 구조는 복사한 것처럼 동일해야 한다.
- 바뀌는 것은 텍스트와 숫자뿐이고, 위치 / 크기 / 라운드 / 순서는 모두 고정이다.


11. 코스 탑뷰 이미지 배치 규칙

사용자는 코스 탑뷰 이미지 1장을 제공한다.
코스 탑뷰 이미지에는 사용자가 직접 표시한 기준점이 포함될 수 있다.

화이트 티 표시:
- 화이트 티 위치는 흰색으로 채워진 원으로 표시한다.
- 흰색 원에는 검은색 외곽선 1~2px를 넣는다.
- 이 표시는 화이트 티 위치이자 거리 측정 시작점이다.

검은색 X 표시:
- 파4 / 파5 홀의 경우 사용자는 추천 타깃 또는 200m 지점에 검은색 X를 표시할 수 있다.
- 검은색 X는 화이트 티 기준 추천 타깃 또는 200m 지점이다.
- 고도차를 반영해 사용자가 판단한 실제 공략 기준점이다.

코스 탑뷰 배치:
- 코스 탑뷰 이미지는 좌측 대형 패널 안에 배치한다.
- 원본 비율을 유지한다.
- 이미지를 임의로 늘리거나 왜곡하지 않는다.
- 기본 배치 방식은 contain 방식이다.
- 코스 전체가 잘리지 않도록 한다.
- 남는 여백은 배경색 또는 자연스러운 아이보리 영역으로 처리한다.
- 이미지 안에 있는 흰색 원과 검은색 X는 그대로 보이게 유지한다.

허용되는 보정:
- 밝기/대비 조정
- 불필요한 흰 여백 최소화
- 이미지 가장자리 정돈
- 패널에 맞춘 크기 조정

단, 보정 과정에서도 다음을 지켜야 한다.
- 흰색 원 위치를 바꾸지 않는다.
- 검은색 X 위치를 바꾸지 않는다.
- 흰색 원과 검은색 X를 삭제하지 않는다.
- 흰색 원과 검은색 X를 새 디자인 요소로 대체하지 않는다.

금지되는 작업:
- 코스 형태 재창작
- 벙커 위치 변경
- 워터 해저드 위치 변경
- 카트길 위치 변경
- 그린 위치 변경
- 없는 해저드 추가
- 홀 방향 임의 회전
- 코스맵을 새로 그리는 행위
- 흰색 원 위치 변경
- 검은색 X 위치 변경
- 흰색 원 또는 X를 타깃 마크로 대체하는 행위


12. 화이트 티와 검은색 X 처리 규칙

기존에는 AI가 흰색 원과 검은색 X를 인식해서 랜딩존을 생성하도록 했지만, 실제 작업에서 위치가 자주 틀어졌다.

따라서 앞으로는 다음 규칙을 적용한다.

AI 작업:
- 흰색 원을 그대로 보존한다.
- 검은색 X를 그대로 보존한다.
- 흰색 원을 다른 위치로 옮기지 않는다.
- 검은색 X를 다른 위치로 옮기지 않는다.
- 흰색 원을 TEE 0m 마크로 바꾸지 않는다.
- 검은색 X를 타깃 마크로 바꾸지 않는다.
- 흰색 원과 X를 확대하거나 축소하지 않는다.
- 최종 이미지에서 원본에 찍힌 상태 그대로 보이게 둔다.

AI가 하지 말아야 할 작업:
- TEE 0m 라벨 생성 금지
- 타깃 마크 생성 금지
- 200m 텍스트 생성 금지
- 랜딩존 점선 원 생성 금지
- 화이트 티에서 X까지 연결선 생성 금지
- 가이드라인 생성 금지

사용자 후처리:
- 랜딩존 점선 원
- 타깃 마크
- 200m 텍스트
- 화이트 티 기준선
- 기타 거리 관련 표식

위 요소들은 사용자가 최종 이미지 완성 후 직접 추가한다.


13. 파4 / 파5 홀 처리 규칙

파4 / 파5 홀에서는 사용자가 코스 탑뷰 이미지 위에 검은색 X를 표시할 수 있다.

AI 작업:
- 검은색 X를 그대로 유지한다.
- 검은색 X 주변에 타깃 마크를 만들지 않는다.
- 검은색 X 주변에 200m 텍스트를 만들지 않는다.
- 검은색 X 주변에 랜딩존 점선 원을 만들지 않는다.
- 검은색 X를 삭제하거나 대체하지 않는다.

AI는 공략 말풍선과 위험 구역 안내만 추가한다.


14. 파3 홀 처리 규칙

파3 홀에서는 사용자가 검은색 X를 제공하지 않아도 된다.

AI 작업:
- 코스뷰에 200m 텍스트를 표시하지 않는다.
- 코스뷰에 랜딩존 점선 원을 표시하지 않는다.
- 코스뷰에 타깃 마크를 표시하지 않는다.
- 파3의 추천 공략 지점은 우측 하단 그린 분석 영역에만 표시한다.

그린 분석 영역에서의 파3 처리:
- 연녹색 점선 타원으로 추천 공략 구역을 표시할 수 있다.
- 필요 시 붉은색 점선으로 주의 구역을 표시할 수 있다.
- 단, 평탄하거나 특별한 위험 요소가 없는 그린에는 억지로 위험 구역을 만들지 않는다.
- 추천 공략 구역을 넓고 깔끔하게 표시해도 된다.


15. 공략 말풍선 규칙

코스맵 말풍선은 최대 3개만 사용한다.

말풍선 유형:
- 일반 공략: 다크그린 #0A2F1D
- 위험 구역: 다크그린 #0A2F1D
- 치명적 위험: 오렌지 #F2994A + ⚠️

문구 길이:
- 한 말풍선은 최대 2줄
- 한 줄은 10~12자 내외
- 긴 문장 금지

추천 문구 구조:
1번: 티샷 공략
예) 티샷 중앙 공략
예) 우측 센터 공략
예) 좌측 공간 공략

2번: 주요 위험 구역
예) 좌측 워터 주의
예) 우측 OB 주의
예) 우측 PA 주의

3번: 그린 주변 주의
예) 그린 앞 벙커 주의
예) 그린 좌측 안착
예) 앞뒤 경사 주의

중요:
- 말풍선을 4개 이상 넣지 않는다.
- 텍스트가 많아져 코스맵을 가리지 않도록 한다.
- 동일한 내용을 반복해서 쓰지 않는다.
- 말풍선이나 화살표가 흰색 원과 검은색 X를 가리지 않도록 한다.


16. 고도 단면도 처리 규칙

사용자는 고도 단면도 이미지 1장을 제공한다.

중요 원칙:
- 고도 정보는 사용자가 별도로 기록하지 않는다.
- AI는 제공된 경사도 이미지 안의 정보를 그대로 사용한다.
- AI가 고도차를 임의로 추정하거나 새로 만들지 않는다.

배치 방식:
- 우측 상단 패널에 배치한다.
- 원본 비율을 유지한다.
- 불필요한 넓은 흰 여백은 정리한다.
- 고도 그래프의 형태와 수치는 왜곡하지 않는다.
- 그래프 자체를 패널 안에서 가능한 크게 배치한다.

요약 탭:
- 별도의 하단 수치 요약 탭은 만들지 않는다.
- 이유: 고도 단면도 이미지 자체에 충분한 시각 정보가 있으므로, 그래프를 더 크게 보여주는 것을 우선한다.

허용되는 정리:
- 흰 여백 제거
- 그래프 주변 정돈
- 텍스트 선명도 보정
- 패널 안에서 보기 좋게 크기 조정

금지되는 작업:
- AI가 고도 수치를 임의로 수정
- IP 위치를 새로 계산
- 오르막/내리막을 반대로 해석
- 경사 그래프를 새로 창작
- 고도 수치를 임의로 추가


17. 그린 등고선 이미지 처리 규칙

사용자는 그린 등고선 / 그린 경사 이미지 1장을 제공한다.

사용자 제공 이미지 특성:
- 다른 화면에서 잘라온 이미지일 수 있다.
- 일부 글자, 앱 UI, 워터마크, 잘린 글자, 점수 정보 등이 포함될 수 있다.

AI 작업:
- 불필요한 글자, 앱 UI, 점수 정보, 워터마크, 잘린 텍스트를 제거하거나 보이지 않게 정리한다.
- 그린 형태와 색상 정보는 유지한다.
- 그린 주변을 감싸고 있는 모래 벙커의 형태는 훼손하지 않는다.
- 원본 비율은 유지한다.
- 그린 전체가 잘리지 않게 contain 방식으로 배치한다.
- 필요하면 가장자리 불필요 영역만 crop 한다.

가장 중요한 원칙:
- 그린 등고선 이미지를 생성형으로 다시 그리지 않는다.
- 반드시 사용자가 제공한 원본 그린 이미지를 그대로 삽입한다.
- 필요한 경우 crop, 마스킹, 밝기 보정만 수행한다.
- 그린 위에 경사 화살표, 추천 구역, 절대 기피 구역만 새로 추가한다.

금지되는 작업:
- 그린 색상 높낮이 의미 변경
- 그린 형태 왜곡
- 그린을 새로 그려 원본과 다르게 변경
- 벙커 위치 변경
- 벙커 형태 훼손
- 경사 방향을 반대로 해석


18. 그린 색상 해석 규칙

그린 등고선 이미지는 아래 기준으로 해석한다.

- 빨강 / 주황 = 가장 높은 곳
- 노랑 = 높은 곳
- 초록 = 중간
- 파랑 = 낮은 곳

경사 화살표:
- 높은 곳에서 낮은 곳 방향으로 그린다.
- 색상: 흰색
- 투명도: 60~80%
- 두께: 3~5px
- 개수: 2~3개
- 구역별 흐름을 나누어 표시한다.

방향 기준:
- 그린 이미지의 방향은 기본적으로 제공된 이미지 방향을 따른다.
- 별도 정보가 없을 경우:
  - 이미지 상단 = 그린 뒤쪽
  - 이미지 하단 = 그린 앞쪽
  - 이미지 왼쪽 = 좌측
  - 이미지 오른쪽 = 우측
- 사용자가 그린 앞/뒤 방향을 별도로 지정하면 사용자 지정 방향을 우선한다.


19. 그린 분석 표시 요소

필수 요소:
1. 그린 등고선 이미지
2. 주요 경사 화살표
3. 우측 분석 탭 3개
4. 추천 공략 지점
5. 필요 시 절대 기피 지점

우측 분석 탭:
- 인위적인 번호나 숫자 라벨을 넣지 않는다.
- 1번, 2번 같은 넘버링은 제외한다.
- 실전 공략에 필요한 세부 지형/브레이크 특징 3개만 나열한다.

탭 예시:
- 좌측 후방 높음
- 우측 전면 낮음
- 좌→우 브레이크
- 언듈레이션 큼
- 우측·뒤쪽 위험
- 좌측 안착 추천

공략 구역 도식화:
- 그린 위에는 연녹색 점선으로 추천 공략 지점을 표시한다.
- 위험하거나 피해야 할 곳은 붉은색 점선으로 절대 기피 지점을 표시한다.
- 단, 등고선이 평탄하거나 특별한 위험 요소가 없는 무난한 그린일 경우 억지로 위험 구역을 표시하지 않는다.
- 그 경우 추천 공략 구역만 넓게 표시한다.

추천 공략 지점:
- 연녹색 점선 박스 또는 타원
- 중앙 또는 상대적으로 안전한 퍼트 라인 쪽

절대 기피 지점:
- 붉은색 점선 박스 또는 타원
- 급경사, 낮은 구역, 벙커/해저드 인접 구역


20. 범례 및 하단 TIP

범례:
- 간단한 아이콘과 짧은 텍스트로 구성한다.
- 너무 많은 범례를 넣지 않는다.
- 코스뷰 하단 또는 전체 하단에 작게 배치한다.

범례 예시:
- 추천 구역
- 주의 구역
- 벙커
- 워터 / PA
- OB

하단 TIP:
- 가능하면 한 줄로 짧게 작성한다.
- 너무 긴 전략 문장을 넣지 않는다.
- 실제 플레이에 바로 도움이 되는 문장으로 작성한다.

예시:
- 무리한 공략보다 안전한 투퍼트 운영
- 우측 위험을 피하고 중앙~좌측 공략
- 내리막 거리감과 그린 뒤쪽 주의

- 그린 경사 색상 의미를 바꾸지 않는다.

25. 최종 프롬프트용 핵심 문장

사용자가 제공한 코스 탑뷰, 그린 등고선, 고도 단면도 이미지를 새로 그리지 말고, 원본 형태를 유지한 채 고정 템플릿에 배치하고 주석만 추가해 주세요.

전체 캔버스는 1000 x 1414 px로 고정합니다.
모든 홀은 같은 레이아웃을 사용해야 합니다.

상단 헤더는 절대 좌표로 고정합니다.
코스명, 홀 번호, PAR, 티박스 거리 박스 4개, 나침반은 모든 홀에서 같은 위치에 있어야 합니다.
바뀌는 것은 텍스트와 숫자뿐입니다.

코스명은 x=42, y=40 위치에 좌측 정렬합니다.
홀/PAR은 x=42, y=78 위치에 좌측 정렬합니다.
티박스 거리 박스 그룹은 x=420, y=72 위치에 4개 한 줄로 고정합니다.
나침반은 x=850, y=36 영역에 고정합니다.

전체 외곽 라운드는 12px, 내부 패널 라운드는 12px, 티박스 박스 라운드는 8px로 고정합니다.
라운드를 임의로 더 크게 만들지 마세요.

나침반은 N을 항상 위쪽에 고정합니다.
나침반 전체를 회전하지 마세요.
N 글자를 회전하지 마세요.
사용자가 제공한 방위각은 홀 진행 방향이 북쪽 기준 시계방향 몇 도인지를 뜻합니다.
회전하는 것은 홀 진행 방향 화살표뿐입니다.
방위각 150도라면 화살표는 오른쪽 아래, 약 5시 방향을 향해야 합니다.
방위각 180도라면 화살표는 아래, 6시 방향을 향해야 합니다.

코스 탑뷰 이미지는 좌측 패널에 원본 비율 유지 contain 방식으로 배치합니다.

코스 탑뷰 이미지 안에는 사용자가 표시한 기준점이 있을 수 있습니다.

1. 흰색으로 채워진 원 + 검은색 외곽선
   - 화이트 티 위치입니다.
   - 이 표시를 삭제하거나 이동하지 마세요.
   - 이 표시를 TEE 0m 마크로 바꾸지 마세요.
   - 최종 이미지에 그대로 보존해 주세요.

2. 검은색 X
   - 파4 / 파5 홀에서 제공될 수 있습니다.
   - 화이트 티 기준 추천 타깃 또는 200m 지점입니다.
   - 이 표시를 삭제하거나 이동하지 마세요.
   - 이 표시를 타깃 마크로 바꾸지 마세요.
   - 최종 이미지에 그대로 보존해 주세요.

AI는 코스뷰에 타깃 마크, 200m 텍스트, 랜딩존 점선 원, TEE 0m 라벨, 화이트 티 기준선, 가이드라인을 추가하지 마세요.
이 요소들은 사용자가 최종 이미지 완성 후 직접 추가합니다.

파3 홀에서는 검은색 X를 제공하지 않을 수 있습니다.
파3 홀에서도 코스뷰에 200m 텍스트, 랜딩존 점선 원, 타깃 마크를 생성하지 마세요.
파3의 추천 공략 지점은 필요 시 그린 분석 영역에 연녹색 점선 추천 구역으로 표시해 주세요.

고도 정보는 제공된 고도 단면도 이미지 안의 수치를 그대로 사용하고, AI가 별도로 고도차를 추정하거나 새로 만들지 마세요.
별도의 고도 요약 탭은 만들지 말고, 고도 단면도 이미지를 더 크게 보여 주세요.

그린 등고선 이미지는 우측 하단에 배치하되, 반드시 사용자가 제공한 원본 이미지를 그대로 삽입해 주세요.
그린 이미지를 생성형으로 다시 그리지 마세요.
잘라온 이미지에 포함된 불필요한 글자, 앱 UI, 점수 정보, 워터마크는 crop 또는 마스킹으로 제거해 주세요.
단, 그린 형태와 주변 벙커 형태는 훼손하지 마세요.

그린 색상은 빨강/주황=높음, 노랑=높음, 초록=중간, 파랑=낮음으로 해석하고, 흰색 화살표는 높은 곳에서 낮은 곳 방향으로 2~3개 그려 주세요.

그린 분석 탭은 숫자 없이 3개만 사용하고, 실전 공략에 필요한 지형/브레이크 특징만 짧게 작성해 주세요.

그린 위에는 연녹색 점선으로 추천 공략 지점을 표시하고, 필요 시 붉은색 점선으로 절대 기피 지점을 표시해 주세요.
단, 평탄하거나 특별한 위험 요소가 없는 무난한 그린에는 억지로 위험 구역을 만들지 마세요.

말풍선은 최대 3개만 사용하고, 각 문구는 2줄 이하로 짧게 작성합니다.
말풍선과 화살표가 흰색 원과 검은색 X를 가리지 않도록 배치해 주세요.
레이아웃, 색상, 글자 크기, 범례 위치는 모든 홀에서 동일하게 유지해 주세요.


26. 실전 작업 시 주의할 점

이미지 생성 AI는 참고 이미지를 그대로 배치하기보다 비슷하게 다시 그리는 경향이 있다.

따라서 정확도가 중요한 코스맵, 그린 등고선, 고도 단면도는 이미지 생성 방식보다 원본 이미지를 그대로 캔버스에 올리고 주석만 합성하는 방식이 적합하다.

특히 그린 이미지는 반드시 원본을 유지해야 한다.
그린 형태, 색상 분포, 벙커 위치가 바뀌면 잘못된 공략 정보가 된다.

또한 화이트 티와 검은색 X의 위치는 AI가 정확히 유지하지 못할 수 있다.
따라서 AI가 이 지점을 인식해 랜딩존을 만들게 하지 않는다.
화이트 티, X, 랜딩존, 200m 표시는 사용자가 직접 후처리하는 것이 더 정확하다.

앞으로 종합 이미지를 만들 때는 다음 방식이 가장 좋다.

1. 원본 코스맵을 좌측 패널에 그대로 배치
2. 원본 고도 단면도를 우측 상단에 그대로 배치
3. 원본 그린 등고선을 우측 하단에 그대로 배치
4. 코스뷰의 흰색 원과 검은색 X는 원본 그대로 보존
5. AI는 코스뷰에 공략 말풍선과 위험구역 안내만 추가
6. AI는 코스뷰에 랜딩존, 타깃, 200m 텍스트, TEE 0m 라벨, 기준선을 추가하지 않음
7. 그린 분석 영역에만 경사 화살표, 추천구역, 기피구역 추가
8. 사용자가 최종 이미지 완성 후 직접 랜딩존과 거리 표식을 추가
